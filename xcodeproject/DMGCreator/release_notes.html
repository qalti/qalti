What's New in Qalti 0.5.9

  • Video recording in CLI and GUI - Record test execution videos in both command-line and graphical interfaces. Toggle recording on/off via the Recording menu in GUI or use --record-video flag in CLI. Optionally auto-delete videos from successful test runs to save disk space
  • Gemini 3 model support - Added support for Google's Gemini 3 Pro and Gemini 3 Flash preview models with optimized reasoning configuration for better test generation and execution
  • GPT-5 model support - Added support for OpenAI's GPT-5, GPT-5 Mini, and GPT-5 Nano models, expanding your AI model options for test automation
  • Fixed Allure report issues - Resolved screenshot filename collisions in Allure reports that occurred when running multiple tests consecutively. Screenshots now use unique datetime prefixes to prevent overwrites
  • Allure reports in scheduler runs - Scheduler now fully supports Allure report generation. Use --allure-dir flag to generate comprehensive Allure reports for parallel test executions
  • Execution time tracking - Test execution duration is now accurately tracked and included in Allure reports, providing better insights into test performance and timing
  • Improved simulator opening speed - Enhanced simulator boot and connection reliability with optimized timing, reducing wait times when launching simulators for test execution
